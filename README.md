# Progress

Прототип блока Progress для мобильных web-приложений: кольцевой индикатор
выполнения процесса с API для управления состоянием.

Технологии: чистые HTML / CSS / JS (ES-модули) — без библиотек,
фреймворков, шаблонизаторов и шага сборки.

## Структура

```
index.html        — демо-приложение
css/styles.css    — стили демо-приложения
js/progress.js    — переиспользуемый компонент Progress (самодостаточный)
js/app.js         — привязка контролов демо к API компонента
```

Компонент — один файл `js/progress.js`: он сам инжектит свои стили,
поэтому для переиспользования в другом приложении достаточно
импортировать один модуль.

## Состояния

| Состояние | Описание |
|---|---|
| **Normal** | Значением `Value` (0–100) задаётся размер дуги. Начало дуги — «12 часов», конец при увеличении значения движется по часовой стрелке и замыкает кольцо при 100. |
| **Animated** | Независимое состояние: блок вращается по часовой стрелке с постоянным периодом. |
| **Hidden** | Скрывает блок со страницы. |

## API

```js
import { Progress } from './js/progress.js';

// Создание: контейнер + начальное состояние
const progress = new Progress(document.getElementById('progress'), {
  value: 60,        // 0..100
  animated: false,
  hidden: false,
  size: 120,        // px
  strokeWidth: 10,  // px
});

// Управление — через свойства-аксессоры
progress.value = 80;      // дуга 80% (ввод за пределами 0..100 приводится к границам)
progress.value;           // 80
progress.animated = true; // включить вращение
progress.hidden = true;   // скрыть блок
progress.destroy();       // удалить блок из DOM
```

Каждое изменение состояния порождает событие `progress:change`
(CustomEvent, всплывает с корневого элемента) с `{ value, animated,
hidden }` в `detail` — демо использует его, чтобы контролы оставались
синхронными, даже когда блок меняют из консоли: экземпляр демо доступен
как `window.progress`.

## Демо-приложение

- **Value** — текстовый ввод числа 0–100 (нецифровые символы
  отбрасываются, значение ограничивается диапазоном);
- **Animate** — переключатель вращения;
- **Hide** — переключатель видимости.

Раскладка адаптируется под ориентацию экрана: в портретной ориентации
кольцо сверху, контролы снизу; в ландшафтной — кольцо слева, контролы
справа.

## Запуск локально

Проект использует ES-модули, поэтому его нужно открывать по HTTP, а не
через `file://`. Любой статический сервер подойдёт, например:

```bash
npx serve .
```

Затем открыть выданный адрес (обычно `http://localhost:3000`). На
GitHub Pages модули работают из коробки.
